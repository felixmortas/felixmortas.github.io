Generate L10n – AI-Powered Flutter Localization VSCode Extension

Simplify and automate localization of Dart/Flutter projects directly within VSCode using AI/LLMs.

Overview

Generate L10n is a VSCode extension designed to streamline the localization workflow for Flutter/Dart projects. It allows developers to automatically update .dart files and generate .arb translation files using AI models like Mistral, OpenAI, or Google LLMs, directly from VSCode without leaving the editor.

This extension is ideal for Flutter developers aiming to maintain multilingual applications efficiently while leveraging AI for generating translations.

Key Features

  • Interactive tree view of /lib Dart files with check/uncheck functionality.
  • Selection of multiple ARB localization files for multi-language support.
  • Input your API key to connect to the AI provider of your choice.
  • Automatic execution of flutter gen-l10n after processing.
  • Native VSCode integration: activity bar icon, command palette commands, tree view.
  • Support for multiple LLM providers and models.

Technologies Used

  • TypeScript – core language for VSCode extension development.
  • VSCode Extension API – for commands, tree views, terminal integration.
  • auto-l10n-ts – library to process Dart and ARB files programmatically.
  • LLM APIs – Mistral, OpenAI, Google for automated translation generation.
  • Flutter SDK – to generate localization files via flutter gen-l10n.

How It Works

  1. Open a Flutter project in VSCode.
  2. Click the L10n Generator icon in the activity bar.
  3. Select the Dart files to process and choose the target ARB files.
  4. Enter your API key for the chosen LLM provider.
  5. Press Play ▶ or run Process Selected Files.
  6. The extension modifies the Dart/ARB files and executes flutter gen-l10n.
  7. Receive a success/failure notification directly in VSCode.

Architecture & Technical Details

  • Modular design: separate modules for tree view, commands, and file processing.
  • Tree view displays /lib folder hierarchy and allows checking files for processing.
  • Commands: toggleCheck, processSelectedFiles, configureExtension, refreshView.
  • Integration with LLMs for generating translation keys and values.
  • Terminal execution for Flutter localization commands.

Challenges & Solutions

  • Challenge: Seamlessly integrating AI-based translation into the VSCode workflow.
    Solution: Abstracted all file processing in auto-l10n-ts and connected commands to tree view events.
  • Challenge: Providing a 1-click experience without external setup.
    Solution: Built full UI in VSCode with API key configuration and automatic terminal execution.

Benefits

  • Saves developers hours of manual localization work.
  • Ensures consistent translation generation across multiple languages.
  • Enables rapid integration of AI-powered localization directly from the editor.
